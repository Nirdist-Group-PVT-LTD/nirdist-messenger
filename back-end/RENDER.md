# Render Deployment

This backend is deployed from the monorepo root with the Render Blueprint at `render.yaml` and the Dockerfile in `back-end/Dockerfile`.

## Service Layout

- `render.yaml` defines one web service and one Render Postgres instance.
- `back-end/Dockerfile` builds the Spring Boot jar and runs it on the Render `PORT`.
- `back-end/src/main/java/com/nirdist/controller/HealthController.java` exposes `/api/health` for Render health checks.

## Environment Variables

| Name | Source | Purpose |
| --- | --- | --- |
| `JDBC_DATABASE_URL` | Render Postgres `connectionString` | Database URL for Spring Boot. The app prepends `jdbc:` automatically, so Render should supply the raw `postgresql://...` value. |
| `DB_USER` | Render Postgres `user` | Database username. |
| `DB_PASSWORD` | Render Postgres `password` | Database password. |
| `JWT_SECRET` | Generated by Render | HS512 signing key for backend JWTs. |
| `JWT_EXPIRATION` | Optional override | JWT lifetime in milliseconds. Defaults to one day. |
| `FIREBASE_SERVICE_ACCOUNT_JSON` | Manual secret value | Firebase Admin service account JSON for OTP verification. Use a Render secret env var. |
| `FIREBASE_CREDENTIALS_PATH` | Optional manual value | Path to a mounted Firebase credentials file, if you prefer file-based credentials over inline JSON. |
| `FIREBASE_PROJECT_ID` | Optional manual value | Overrides the Firebase project id when needed. |
| `PORT` | Render-provided | HTTP port for the container. |

## Notes

- Flyway runs automatically at startup, so the Render Postgres instance is migrated as soon as the service boots.
- If you use `FIREBASE_CREDENTIALS_PATH`, you also need to provide the credential file yourself; the default Render setup is to store the JSON in `FIREBASE_SERVICE_ACCOUNT_JSON`.
- The backend health check is `/api/health`.

## Local Docker Build

```bash
docker build -t nirdist-backend back-end
docker run --rm -p 8080:8080 -e JDBC_DATABASE_URL=postgresql://localhost:5432/nirdist nirdist-backend
```
